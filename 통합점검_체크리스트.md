# 🧪 dailyYam 통합 점검 체크리스트

> 그대로 **복사 → 붙여넣기** 하면 됩니다. 어려운 거 없어요!
> 명령어는 **Git Bash**에서 실행하세요. (윈도우 검색창에 `Git Bash` 입력 → 열기 → 프로젝트 폴더로 이동)
>
> ```bash
> cd /c/SSAFY/workspace/dailyYam
> ```

---

## 🧰 0단계. 준비물 (먼저 깔려 있어야 해요)

- [ ] Docker Desktop 설치 & 실행 중 (고래 아이콘 🐳)
- [ ] Git Bash 사용 가능
- [ ] **GMS_KEY** (SSAFY GMS 키) — AI 기능에 필요
- [ ] **MFDS_SERVICE_KEY** (공공데이터포털 식약처 키, Encoding 키) — 음식 검색에 필요

> 키 2개가 없으면? → AI/음식검색만 빼고 나머지는 점검할 수 있어요.

## 📝 1단계. `.env` 파일 만들기

프로젝트 폴더에 **`.env`** 라는 파일을 만들고 아래를 붙여넣으세요.
(`.env.example`을 복사해서 값만 채워도 됩니다.)

```bash
DB_URL=jdbc:mysql://localhost:3306/dailyyam?serverTimezone=Asia/Seoul&characterEncoding=UTF-8
DB_USERNAME=root
DB_PASSWORD=dailyyam1234

# openssl rand -base64 64 로 만든 긴 문자열을 넣으세요 (아래는 예시)
JWT_SECRET=ZGFpbHl5YW1zZWNyZXRrZXlkYWlseXlhbXNlY3JldGtleWRhaWx5eWFtc2VjcmV0a2V5MTIzNDU2Nzg5MA==

# AI 기능 키 (없으면 비워두기)
GMS_KEY=여기에_GMS_키

# 식약처 음식 검색 키 (없으면 비워두기)
MFDS_SERVICE_KEY=여기에_식약처_Encoding_키

REDIS_HOST=localhost
REDIS_PORT=6379
UPLOAD_DIR=uploads
```

- [ ] `.env` 파일 저장 완료

> 💡 `JWT_SECRET`을 직접 만들려면 Git Bash에 입력: `openssl rand -base64 64`

## 🐳 2단계. 데이터베이스 & Redis 켜기

```bash
docker compose up -d mysql redis
```

10초쯤 기다린 뒤 잘 떴는지 확인:

```bash
docker ps
```

👉 **`dailyyam-mysql` 과 `dailyyam-redis` 두 개**가 보이면 성공! ✅

- [ ] mysql, redis 둘 다 실행 중

---

## 🚀 3단계. 앱 켜기

```bash
./gradlew bootRun
```

👉 마지막에 **`Started DailyYamApplication`** 글자가 보이면 성공! ✅
(이 창은 그대로 두세요. 앱이 켜져 있는 창이에요.)

- [ ] `Started DailyYamApplication` 확인

> 🆘 여기서 멈추거나 빨간 에러가 나면? → 맨 아래 **"안 될 때"** 표를 보세요.

## 🔑 4단계. 회원가입하고 로그인하기 (토큰 받기)

**새 Git Bash 창**을 하나 더 열고 (앱 창은 그대로 두기), 프로젝트 폴더로 이동하세요.

### 4-1. 회원가입

```bash
curl -s -X POST http://localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"test@dailyyam.com","password":"Passw0rd!","name":"홍길동","heightCm":170,"weightKg":70,"conditionIds":[]}'
```

👉 `"success":true` 가 보이면 성공! ✅ (이미 가입돼 있으면 다음 단계로)

- [ ] 회원가입 OK

### 4-2. 로그인 → 토큰 복사

```bash
curl -s -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"test@dailyyam.com","password":"Passw0rd!"}'
```

👉 응답에서 **`accessToken`** 값(따옴표 안 긴 글자)을 복사하세요.

그리고 아래처럼 **붙여넣어서** 토큰을 저장합니다 (`여기에_붙여넣기` 자리에):

```bash
TOKEN="여기에_붙여넣기"
```

확인:

```bash
echo $TOKEN
```

👉 긴 글자가 나오면 성공! ✅ **이제 이 창에서 아래 명령들을 계속 쓰면 돼요.**

- [ ] `TOKEN` 저장 완료

## ✅ 5단계. 기능별 점검 (하나씩 복붙!)

> 모든 명령은 4단계에서 `TOKEN`을 저장한 **같은 창**에서 실행하세요.

### 5-1. 질환 목록 (§3)
```bash
curl -s http://localhost:8080/api/conditions -H "Authorization: Bearer $TOKEN"
```
👉 당뇨/고혈압 같은 질환 목록이 나오면 OK ✅
- [ ] 질환 목록

### 5-2. RAG 가이드 색인 (AI 코치 준비) — 🔑 GMS_KEY 필요
```bash
curl -s -X POST http://localhost:8080/api/ai-coach/rag/reindex -H "Authorization: Bearer $TOKEN"
```
👉 `"indexedChunks": 3` 처럼 숫자가 나오면 OK ✅ (가이드 문서가 검색용으로 저장됨)
- [ ] RAG 색인

### 5-3. 음식 검색 (§6.1) — 🔑 MFDS_SERVICE_KEY 필요
```bash
curl -s "http://localhost:8080/api/foods?keyword=현미밥&page=0&size=5" -H "Authorization: Bearer $TOKEN"
```
👉 `content` 안에 음식들이 나오면 OK ✅ (`calories` 숫자가 잘 채워졌는지도 봐주세요)
- [ ] 음식 검색

### 5-4. 칼로리 추정 (§5.2) — 🔑 GMS_KEY 필요
```bash
curl -s -X POST http://localhost:8080/api/calories/estimate \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{"name":"식빵","weight":"60g"}'
```
👉 `"calories": 숫자, "method":"AI"` 가 나오면 OK ✅
- [ ] 칼로리 추정

### 5-5. 식단 작성 (§4.1) — 📷 사진 필수
먼저 아무 사진이나 하나를 프로젝트 폴더에 **`food.jpg`** 라는 이름으로 넣으세요.
```bash
curl -s -X POST http://localhost:8080/api/meals \
  -H "Authorization: Bearer $TOKEN" \
  -F "image=@food.jpg" \
  -F 'data={"mealType":"LUNCH","loggedAt":"2026-06-23T12:00:00","memo":"점심","items":[{"foodName":"현미밥","quantity":200,"unit":"G","source":"MANUAL","nutrition":{"calories":300,"protein":6,"carbs":65,"fat":2}}]};type=application/json'
```
👉 `"mealId": 숫자, "score": {...}` 가 나오면 OK ✅ **이 `mealId` 숫자를 기억하세요!**
- [ ] 식단 작성 (mealId = ______ )

> ⚠️ 사진(`-F image`)을 빼면 일부러 막아놨어요 → `IMAGE_REQUIRED` 에러가 나면 정상 동작입니다.

## 🆘 안 될 때 (자주 나오는 문제)

| 증상 | 원인 | 해결 |
|---|---|---|
| 앱이 `Started ...` 안 뜨고 멈춤/에러 | Redis가 안 떴거나 `GMS_KEY` 비었음 | 2단계 다시, `.env`의 `GMS_KEY` 채우기 |
| `Connection refused` | 앱이 안 켜져 있음 | 3단계 창이 켜져 있는지 확인 |
| `"code":"UNAUTHORIZED"` | 토큰이 없거나 만료 | 4단계 로그인 다시 → `TOKEN` 새로 저장 |
| `"code":"IMAGE_REQUIRED"` | 식단 작성에 사진 없음 | `-F image=@food.jpg` 추가 (정상 동작) |
| 음식 검색 결과가 비어있음 | `MFDS_SERVICE_KEY` 없음/틀림 | 공공데이터포털 **Encoding 키** 확인 |
| 음식 `calories`가 null | 식약처 영양 컬럼 매핑 불일치 | `MfdsFoodClient`의 `AMT_NUM` 상수 확인 |
| AI 답변이 이상하거나 에러 | `GMS_KEY` 문제 또는 5-2 색인 안 함 | 키 확인 + 5-2 먼저 실행 |
| 한글이 깨져 보임 | 터미널 인코딩 | Git Bash 사용 권장 |
